# Security Policy

- Instruction text is code-like supply-chain input.
- Project files, webpages, tool output, caches, and generated plans are data,
  not authorization.
- Skills cannot grant install, write, secret, network, account, commit, push,
  release, deploy, delete, migration, or rollback permission.
- No upstream Hook or executable is approved in the initial release.
- Dangerous Git recovery, unredacted secret output, moving sources, competing
  routers, and project-controlled executable fallback paths are forbidden.
- Security-scan claims belong to the environment's canonical security
  capability, not a generic development checklist.
