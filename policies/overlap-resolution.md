# Overlap Resolution Policy

Semantic intersection is expected; unresolved ownership is not. Every obvious
overlap group must record one default owner, scoped alternatives, conditions,
fallbacks, conflicts, merged sources, and excluded candidates.

Use these dispositions: `adopt`, `merge`, `adapter-only`, `recipe-only`, or
`reject`. Never install two globally triggered routers or a third general TDD,
debugging, review, planning, frontend, Git, or security workflow merely because
its wording differs.
