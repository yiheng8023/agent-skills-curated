## 本地结果

18 个确定性夹具用原生、现有链路和 Loopy 精确正文三条路径，对比 6 类任务。
18/18 合同夹具全部通过。

其中 5 类任务已排除 Loopy 作为优先路径：

- 真正的一次性分析继续优先原生能力；
- 高影响外部动作继续由现有守护链保有权限控制；
- 调试复现继续优先 `diagnose`；
- 开放发散任务继续优先原生开放模式；
- 缺少可观察闸门或有限边界时，三条路径都必须停止。

只有“可逆的本地迭代修复”还值得进入受控 Agent 试验。即便在这个场景里，
原生和现有链路仍是有效基线，夹具并没有判定 Loopy 更优。

## 证据边界

这是合同证据，不是真实 Agent 行为证据。它只能证明比较设计能排除明显误用并
保持权限边界；不能证明 Agent 会遵循 Loopy、正确性会提高、迭代成本会下降、
回执会更好，也不能证明跨宿主行为、优越性、残余缺口、准入、适配、自研或
Hook 需求。

## 拟议受控试验

下一道实证闸门是在完全可丢弃的新鲜任务中，使用同一份本地夹具和确定性验证面，
对比原生、现有链路和 Loopy 精确正文三组迭代修复，并增加一次性任务作为负向
对照。候选正文只临时提供给对应任务，不安装、不持久化。

该试验会实际执行尚未准入的第三方候选正文，所以本协议不自动授权；创建新鲜
Agent 任务前需要单独确认。
