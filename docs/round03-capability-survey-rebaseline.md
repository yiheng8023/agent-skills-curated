# Round 03 Capability-Survey Rebaseline — Owner Review Required

Date: 2026-07-15

Round 03 remains inactive. Its former adaptation-and-admission plan overlaps
work already completed through separately approved Round 02 follow-on gates.
This proposal replaces that immediate execution direction with an evidence-first
capability survey and residual-gap proof phase.

The read-only STM, P, and SG inputs are now source-pinned by
`registry/round03-demand-coordinate-source-contract.json`. That contract fixes
source identity, research authority state, evidence vocabulary, and the
promotion firewall without copying the external research bodies. Demand-record
extraction remains future Round 03 work after owner activation.

## Proposed Decision Chain

```text
bind evidenced STM / P / SG demand coordinates and limits
-> measure dated native / official / runtime / installed coverage
-> run bounded read-only public metadata discovery
-> cluster, deduplicate, and select minimum representatives
-> pin sources and deep-review without executing candidate code
-> compare native / single-Skill / composition / non-Skill Harness paths
-> classify covered, supported residual gap, or unproven gap
-> if a residual gap is supported, compare Skill-only / Skill + Hook /
   Hook or other non-Skill control / project standard / human checkpoint
-> produce the ten-part result package and next-round candidates
```

Discovery stops when additional candidates no longer materially change
coverage, architecture alternatives, dispositions, or residual-gap evidence.
Remaining uncertainty and recheck triggers must be recorded.

Hook is considered only after a material residual gap is supported. It is
optional, not a default attachment to every repository-authored Skill. A Hook
is suitable only when event-driven recall or a bounded pre/post check is needed,
the host surface is inspectable and reversible, and measured benefit exceeds
noise, latency, permission, context, and maintenance cost. It remains a
consumer-owned recall or enforcement aid, never an authority or evidence source.

## Boundaries

- Public GitHub metadata and minimum necessary source text may be read after
  this rebaseline is accepted.
- No candidate code, script, dependency, Hook, command, or network-capable
  payload may be executed.
- No account connection, OAuth, credential, payment, private-data expansion,
  live runtime mutation, cross-repository write, Skill admission, release
  mutation, standards promotion, or remote push is authorized.
- Codex-only or weak-model evidence must not be presented as universal
  cross-Agent proof.

## Required Result

The stage must deliver all ten artifacts already mapped by the program plan:
native/runtime baseline; clustered candidate inventory; STM/P/SG matrix;
single and composed alternatives; dispositions; residual gaps; evidence limits;
recommendations for the three experimental collaboration Skills and Hooks;
discovery stopping and refresh evidence; and an explicit non-authorization
statement.

Owner review is required before this plan can activate Round 03 discovery.
