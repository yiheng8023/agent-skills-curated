# 用户主权与外部管理能力共存对账——2026-07-18

本对账验证本仓的用户主权合同，同时让多 Agent 外部状态共存继续保持部分完成。

## 已验证的用户主权合同

用户安装、Agent 安装、项目本地、生态管理和所有权未知的能力，默认都属于外部所有。
发现、下载、准入或同名冲突都不代表接管、执行、安装或替换。用户可以自由修改；但
修改后的衍生版本不再自动继承原始精选摘要和验证结论。

历史可丢弃根目录证据演练了所有权检查、外部内容冻结、摘要绑定计划、锁、备份、
追加式日志、应用、验证、回滚和恢复。后续可丢弃 Codex 形态预览枚举两个根目录，
阻止外部所有权，拒绝声明根目录以外的路径，而且不修改目标、不创建持久事务。

本仓当前不假定产品账号或遥测，也不拥有真实消费者操作。运行变更继续由消费者拥有，
必须明确、可预览、可逆。因此 `acceptance.user-sovereign-capability-governance` 在本仓
治理和只读证据范围内可以升级为已验证。

当前只读观测又补充了一种真实共享目录形态：43 个目标解析到 CC Switch，27 个目录由
Lark 锁管理，3 个逐目录匹配消费者仓；没有修改其中任何一个。这证明外部与生态所有权
在真实目录中确实共存，同时也暴露出当前 CC Switch 投影的来源仓记录为 0，尚未实现
来源保真。

## 仍然部分完成

现有证据覆盖一种可丢弃 Codex 形态、一个已被替代的历史事务核心和一个真实只读共享
目录；但还没有覆盖第二个 Agent、消费者自有转换、带外漂移、多 Agent 冲突、备份回滚
和外部哨兵保留。因此
`acceptance.foreign-managed-capability-coexistence` 继续保持部分完成。

本对账只读了真实 Agent Home 和 CC Switch 数据库，没有读取凭据，也没有修改 Skill、
链接、数据库、Agent 配置、事务、备份或 Hook；Hook 本身没有读取。
