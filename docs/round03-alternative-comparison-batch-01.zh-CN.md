# Round 03 替代路径比较（批次 01）

状态：2026-07-15 已完成不执行候选代码的验证型比较。

四条已选需求链均比较了 9 类路径：原生能力、官方/运行时、单 Skill、
组合 Skill 或 Recipe、已评审外部 Skill、非 Skill Harness/工具、项目硬
标准、人工/领域权威以及条件式 Hook。每条路径都记录支持度、总成本、
失败传播和当前处置。

## 判定

- EL-01 保留“原生能力 + 仓库事实 + handoff + contracts”的现有组合；
  跨载体的中断与重入行为尚未证明。
- EL-02 保留原生优先和规范路由器；未决问题是实时根目录、冲突、可移植
  性与成本证据，并非已经证明缺少一个 Skill body。
- EL-03 不是外部 Skill 缺口；高后果结论仍需适用的、可问责的人工或领域
  权威。
- EL-04 更适合由项目治理数据和验证工具承载，而不是再做一个独立 Skill；
  仍需 source-to-consumer 的夹具图证据。

本批次得到 **0 个已证明残余缺口**、**3 个未证明残余缺口**、**1 个人工/
领域权威边界**。自制 Skill 与 Hook 均不具备入场资格，默认仍然是 no Hook。

## 下一证据门

为 EL-01、EL-02 设计有边界的行为与成本证据，为 EL-04 设计投影夹具：
包括中断/重入配对检查、根目录与同名冲突观测、误触发与上下文成本观测，
以及带类型的 source-to-consumer 生命周期边。这里不包括执行或安装已评审
候选。

Skill、Hook、标准晋升、运行时变更、消费者写入、跨仓交付或远端 push。
