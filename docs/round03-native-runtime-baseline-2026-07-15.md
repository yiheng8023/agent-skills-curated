# Round 03 Native And Runtime Baseline

Date: 2026-07-15

This dated read-only baseline covers the current OpenAI Codex desktop workspace
only. It records local native/runtime metadata before any external substitution
decision and does not generalize the result to other Agents or hosts.

The local surface is already large: four inspected roots contain 422
`SKILL.md` files, 326 distinct directory names, and 87 duplicate-name groups.
The `intent-contract`, `capability-router`, and `closure-contract` pairs under
`.codex` and `.agents` are byte-for-byte identical at observation time. This
proves overlap and current identity, not active-root precedence or behavior.

A candidate advisory Hook policy and Codex host profile also exist locally.
Their own metadata says file presence does not prove activation. No Hook
behavior probe was run, and activation, event delivery, value delta, failure
behavior, and cost remain unverified.

## Routing result

- `EL-01`, `EL-02`, and `EL-04` now have concrete, bounded insufficiency and
  public metadata questions. Read-only discovery may start for those questions.
- `EL-03` is not routed to external Skill discovery because its unresolved
  boundary is accountable human or domain authority, which a general Skill
  cannot supply.

This baseline proves neither behavioral coverage nor a residual gap. It does
not authorize candidate execution, installation, account connection, Hook
enablement, Skill adaptation or authoring, standards promotion,
cross-repository writes, remote push, or publication.
