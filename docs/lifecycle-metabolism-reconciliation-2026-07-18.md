# Lifecycle Metabolism Reconciliation — 2026-07-18

This reconciliation separates a working feedback return path, a deterministic
approved-Skill retirement contract, and still-open live-consumer evidence.

## What is verified

The repository has one observed consumer feedback cycle. The `codex-user-config`
consumer verified a selected batch, after which one candidate remained a Recipe
projection and two were merged into existing approved Skills. No standalone
Skill was added. The result returned to discovery as exact-candidate
deduplication, preserved the upstream repository boundary, and selected
`pause and observe` before another batch.

The lifecycle policy and Harness now bind six event classes: consumer,
community, security, license or provenance, upstream refresh, and validation.
Each can reopen review without automatically replacing the last known-good
release. PM Skills also provides an observed read-only upstream revision-delta
review.

This evidence verifies the repository-governed feedback return path. It does
not claim that every trigger class has occurred in production.

## Deterministic retirement contract

Eight policy fixtures now cover five accepted outcomes—retain, supersede,
deprecate, rollback, and retire—and reject automatic upstream replacement,
retirement without migration and rollback, and feedback-based direct approval.
This verifies that the repository contract supports update, supersession,
deprecation, migration, rollback, and retirement without requiring a live
mutation.

The custom Manager initiative remains non-payload historical evidence. No live
approved-Skill consumer has yet completed this retirement path, so observed
runtime maturity and cross-consumer effectiveness remain open claims even
though the repository policy and deterministic fixture acceptance is verified.

## Authority boundary

A feedback record may reopen review or suppress an exact reproposal. It cannot
approve a candidate, mutate the release, overwrite a consumer, install a Skill,
or change a Hook or runtime. Release and consumer changes retain their separate
authority and verification gates.
