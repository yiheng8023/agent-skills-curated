# Layered Reliability Projection Reconciliation

## Decision

The repository now verifies the layered-reliability and project-standard
precedence contracts at their declared scope: a governance projection, not a
claim of live runtime effectiveness.

The smallest-sufficient sequence is:

```text
instructions and rules
-> Skills and Recipes
-> scripts, schemas, and validators
-> consumer-owned Hooks, CI, or runtime controls
-> project-owned hard standards
-> evidence and accountable human decisions
```

Not every task needs every layer. Native reasoning, no Skill, no Hook, and no
additional structure remain valid outcomes. A stronger layer is added only
after evidence shows a material gap and its benefit exceeds its context,
control, permission, and maintenance cost.

## Precedence Boundary

An admitted project-owned hard standard outranks conflicting generic Skill
guidance because the project authority owns the normative decision. This
repository may preserve evidence and prepare a future non-authoritative
candidate contract, but it does not admit, publish, or install a project hard
standard.

This verifies `acceptance.layered-reliability-model` and
`acceptance.project-standard-precedence`. It does not verify consumer mappings,
cross-host behavior, Hook modes, standard-candidate maturity, or an operational
standard migration cascade.

## Deliberate Deferral

Hard-standard extraction remains deliberately deferred. It requires stable
value across independent sources, Agents or hosts, task classes, and real
feedback cycles, followed by a separate project-authority decision. The
current layers, fields, and review protocols are working hypotheses rather
than premature universal standards.

## Non-Actions

- No Skill, Recipe, Hook, CI, runtime, or standard implementation.
- No consumer, project, release-manifest, or approved-payload mutation.
- No cross-repository write, commit, or push.
