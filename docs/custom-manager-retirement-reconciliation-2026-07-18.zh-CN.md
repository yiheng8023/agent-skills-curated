# 自研 Manager 退役校准 — 2026-07-18

自研 Manager 不再是现行产品目标。在受支持的场景中，CC Switch 可以承担来源、
安装、更新、分发、备份和恢复等运行管理。本仓负责跨 Agent 的 Skill 评审治理、
来源追溯、处置、可移植合同和残余缺口判断；宿主配置与真实运行权威继续属于消费端。

## 当前迁移

现行战略身份从 `production-capability-manager` 迁移为“来源保真的跨 Agent 能力
治理”。五条现行验收要求改用不依赖 Manager 的正向表达：用户主权、外来管理
能力共存、广义能力生态边界、先发现复用后自研，以及可选 Hook 模式。

此前的设计、拓扑、Rust 可丢弃根目录事务实验和 Codex 只读适配器预览继续作为
历史证据保留。其文件名、ID、已接受决策和有界验证不改写、不删除，也不再提供
现行实现权限。

## 重新启动边界

只有证明存在具体的运行管理残余缺口、可行替代路径已经失败、消费端范围明确、
迁移与维护成本已评估，并取得单独的用户决策后，才可以重新讨论自研 Manager。
现有历史工作本身不能成为重新启动的权限。

本次校准不修改 CC Switch 或真实 Agent，不执行候选，不删除历史文件，不跨仓写，
不提交，也不推送。
