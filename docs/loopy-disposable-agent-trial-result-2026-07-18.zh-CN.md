# Loopy 可丢弃 Agent 对照试验结果（2026-07-18）

## 结论

精确固定版本的 Loopy 正文处置为：**仅作参考，不准入**。它在本次有边界试验中
表现正确且克制，但没有证明自己相对原生推理和现有自适应合同链路这两个基线都具有
实质增益。候选正文没有被修改、复制入库、安装，也没有写入真实 Agent、用户配置或
Hook。

## 试验边界

- 候选：`github:Forward-Future/loopy`，固定提交
  `75966cbd572a4185064971c9fe5e9c52e8f8456d`；
- 精确 Skill blob：`5fe3082a41521c1e5793d1a271990bc841c9a92f`；
- 宿主/模型：Windows 上的 Codex CLI 0.144.5、`gpt-5.6-sol`、medium reasoning；
- 三臂：原生、现有自适应链路、Loopy 精确正文；
- 两类场景：可逆本地迭代修复、一次性分析负对照；
- 每格重复两次，共 12 个全新、临时、不持久化的正式 Agent 任务；
- 不允许任务联网、安装、持久化、发布、真实配置或 Hook 改动及外部动作。

第一次启动因共享子 Agent 沙箱被误设为只读、验收命令被阻止而作废。随后增加硬预检，
先证明补丁可写且本地 Python 可执行，才放行正式矩阵。作废尝试只保留哈希化的方法证据，
不参与评分。

## 结果

12/12 次正式运行全部正确；12/12 回执完整、终态诚实、权限边界保持。Loopy 在两次
一次性任务中都明确选择不启用循环，因此循环误判为 0。
按比例路由的现有链路臂在四次运行中都选择了允许的原生/无需 Skill 路径，没有打开
任何已安装合同 Skill 正文；这是实测路由结果，不能表述成这些正文已经执行。

| 场景 | 试验臂 | 正确 | 平均秒数 | 平均命令数 | 平均输入 token | 平均输出 token |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| 迭代修复 | 原生 | 2/2 | 56.901 | 4.0 | 189,311.5 | 1,516.0 |
| 迭代修复 | 现有链路 | 2/2 | 64.153 | 4.5 | 195,651.0 | 1,998.5 |
| 迭代修复 | Loopy 精确正文 | 2/2 | 59.028 | 3.5 | 201,975.5 | 1,867.5 |
| 一次性负对照 | 原生 | 2/2 | 23.311 | 1.0 | 60,370.0 | 461.0 |
| 一次性负对照 | 现有链路 | 2/2 | 24.039 | 1.0 | 60,556.0 | 562.5 |
| 一次性负对照 | Loopy 精确正文 | 2/2 | 22.808 | 1.0 | 68,490.0 | 537.5 |

在迭代修复中，Loopy 比原生慢 3.738%，比现有链路快 7.989%，但输入 token 分别多
6.69% 和 3.233%。在一次性负对照中，它只比原生快 2.158%、比现有链路快 5.121%，
输入 token 却分别多 13.45% 和 13.102%。这没有跨过预先约定的门槛：必须在成本成
比例的前提下，相对**两个**基线都体现实质增益。

## 证据与限制

入库的机器结果只保存聚合指标、哈希、边界和处置；原始 Agent JSONL、最终回执与
可丢弃工作区均留在本地，不进入仓库。正式原始结果大小为 62,808 字节，SHA-256 为
`5608D775FD26F9CBB50A74FBDAEF80D745C3651A7E237CCCBFF351B0EA492D35`。

本次只覆盖一个 Windows 宿主、一个模型、一档 reasoning、两类场景和每格两次重复。
CLI 还会向所有试验臂展示同一份环境 Skill 清单，且没有受支持的清单禁用开关。因此
结论只支持当前有边界的“仅作参考”处置，不证明跨 Agent、跨模型或生产环境优越性。

下一闸门是绑定另一条有来源证据支持的需求线。只有 Loopy 上游发生实质变化，或出现
与本次结论相反的需求证据时，才重新评估它。
