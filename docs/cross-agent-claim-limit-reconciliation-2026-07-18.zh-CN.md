# 跨 Agent 主张边界对账——2026-07-18

本对账验证的是“主张治理”，不是“所有 Agent 行为一致”。

现在有 9 类证据进入显式主张账本，每一项都记录宿主、模型、推理、加载器、激活
方式、权限、工作区、日期和证据类型，同时写清楚它能支持什么、不能支持什么、
反例和重新检查触发器。

账本覆盖：Codex 原生基线、422 个 Skill 文件的根目录枚举、咨询型 Hook 元数据、
105 个确定性路由场景、12 次 Loopy 试验、结构化消费者投影、公共来源预检、一次
历史 Codex 消费者反馈循环，以及生命周期政策夹具。

## 已验证边界

本仓现在可以确定性阻止这些证据被扩大为：所有 Agent 等价、提高模型能力上限、
跨宿主行为已证明、隐式激活已证明、加载优先级已证明、Hook 实时有效、当前消费者
一致、元数据证明质量或准入，以及用模拟结果证明真实运行时行为。

因此 `acceptance.cross-agent-claim-limits` 在“主张防火墙”范围内可以升级为已验证。
这不等于跨 Agent 一致性、当前消费者映射、真实 Hook 行为或生产有效性已经验证；
这些结论仍需要各自宿主与消费者拥有的证据。

## 为什么重要

可移植内容可以集中治理，但运行时真相必须留在本地。同一份 Skill 正文可能同时
存在于多个宿主；然而文件存在、显式加载、隐式激活、宿主消费、模型遵循和最终
行为是不同证据层。主张账本避免把它们压扁成一个过大的结论。
