# 原生与运行时基线证据缺口对账

状态：继续保持 `partial` 才诚实。

2026-07-15 的证据是一个有价值的单宿主元数据基线，绑定了当时 4 条需求记录。它明确了
Windows 上的 Codex desktop、声明的模型家族、不可观察的推理细节、加载器限制、仓库范围
权限、工作区、日期、路径、哈希和非主张边界。其中历史字段
`baselineRecordedForEveryDemandRecord=true` 只表示当时链接批次内的 4 条记录，不是当前
调研的全部记录。

当前调研共有 8 条需求记录和 62 行有界坐标矩阵。EL-05 至 EL-08 都有替代方案评审，
但这些评审不是维度完整的日期化基线：它们没有逐需求分别记录原生、官方、运行时、
已安装、已授权能力的可用性，也没有逐需求的激活模式、可复现探测方法和原始结果。
坐标映射和替代方案清单不能证明实时可用性。

因此该验收继续保持 `partial`。未来升级需要为全部 8 条需求逐一生成宿主特定记录，
包含模型、推理、加载器、激活、权限、工作区、探测方法、原始结果、限制和刷新触发器。
其他 Agent 或宿主必须单独建立基线；当前证据不证明跨宿主行为。
