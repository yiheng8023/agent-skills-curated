# Custom Manager retirement reconciliation — 2026-07-18

The custom Manager direction is no longer a current product objective. CC
Switch may provide operational source, install, update, distribution, backup,
and restore management where it is supported. This repository owns reviewed
cross-Agent Skill governance, provenance, dispositions, portable contracts,
and residual-gap decisions. Consumers retain host configuration and live
runtime authority.

## Current migration

The current strategic identity moves from `production-capability-manager` to
source-preserving cross-Agent capability governance. Five current acceptance
surfaces are expressed without a Manager dependency: user sovereignty,
foreign-managed capability coexistence, the broad capability-ecosystem
boundary, discovery and reuse before authoring, and optional Hook modes.

The prior design, topology, Rust disposable-root transaction experiment, and
Codex read-only adapter preview remain historical evidence. Their filenames,
IDs, accepted decisions, and bounded verification are not rewritten or deleted.
They authorize no current implementation.

## Reactivation boundary

A custom Manager may be reconsidered only after a concrete residual operational
management gap, failed viable alternatives, bounded consumers, migration and
maintenance cost, and a separate owner decision are evidenced. Existing
historical work is not sufficient reactivation authority.

This reconciliation performs no CC Switch or live Agent change, candidate
execution, historical-file removal, cross-repository write, commit, or push.
