# Round 02 Source Intake 2026-07-02

This is the first small source-intake slice after broad coverage planning.
It pins three direct third-party Skill sources for later review:

- `github:kepano/obsidian-skills`
- `github:phuryn/pm-skills`
- `github:alchaincyf/huashu-design`

This intake is deliberately narrow. It records source identity, license
posture, default branch revision, observed Skill count, likely coverage domains,
and review focus areas. It does not approve source text, add release payloads,
change `skills/`, change `release-manifest.json`, install anything locally, or
sync Codex, agents, or cc Switch.

The machine-readable record lives in `registry/source-intake-batches.json`.
The next gate is license/provenance, security, portability, overlap, and
adaptation review. Each candidate may still become adopt, merge, recipe-only,
adapter-only, reference-only, or reject.
