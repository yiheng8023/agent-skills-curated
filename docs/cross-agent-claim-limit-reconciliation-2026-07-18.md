# Cross-Agent Claim-Limit Reconciliation — 2026-07-18

This reconciliation verifies claim governance, not universal cross-Agent
behavior.

Nine evidence classes now have an explicit ledger covering host, model,
reasoning, loader, activation, permission, workspace, date, and evidence class.
Each entry records what it supports, what it does not support, a counterexample,
and a recheck trigger.

The ledger covers the dated Codex native baseline, 422-file Skill-root
enumeration, advisory Hook metadata, 105 deterministic routing scenarios, the
12-run Loopy trial, the structural consumer projection, public source
preflight, one historical Codex consumer feedback cycle, and lifecycle policy
fixtures.

## Verified boundary

The repository now deterministically prevents these evidence classes from
becoming claims of universal Agent equality, model-ceiling improvement,
cross-host behavior, implicit activation, loader precedence, live Hook
effectiveness, current consumer parity, metadata-derived quality or admission,
or runtime behavior from simulation alone.

Therefore `acceptance.cross-agent-claim-limits` is verified at the claim
firewall scope. This does not verify cross-Agent parity, current consumer
mapping, live Hook behavior, or production effectiveness. Those claims still
require their own host- and consumer-owned evidence.

## Why this matters

Portable content can be governed centrally while runtime truth remains local.
The same Skill body may be present on several hosts, but presence, explicit
loading, implicit activation, consumption, compliance, and downstream behavior
are different evidence layers. The ledger keeps those layers from collapsing
into one oversized claim.
