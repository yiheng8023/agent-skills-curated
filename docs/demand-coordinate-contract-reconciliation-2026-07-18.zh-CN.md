# 需求坐标合同对账

状态：有界需求坐标合同已验证；需求模型仍然开放。

旧的 `partial` 结论仍写着“需求记录尚不存在”。当前仓库证据已经包含
`EL-01` 至 `EL-08` 共 8 条来源支持的需求记录。每条记录都绑定了 STM、P、SG
坐标、来源证据、明确的证据状态、保留主张、不确定性、受影响主体和复查触发器；
任何残余缺口、Skill、Hook 或标准主张之前，原有晋升防火墙继续生效。

结果包覆盖有界坐标包络全部 62 行：26 个 STM、24 个 P、12 个 SG。每一行都至少
具有一条需求绑定、处置、验证方法、限制和复查触发器。因此，本次验证的是有界合同，
并移除了“记录不存在”这一过时阻塞理由。

这不等于需求模型穷尽。需求记录抽取仍未完成，完整需求模型收口为否，调研收口为否，
也不宣称生态完整。即使有界包络内受支持残余缺口为 0，也不授权执行候选、自研 Skill
或 Hook、沉淀硬标准、改运行时或消费者、提交或推送。

因此，只有 `acceptance.demand-coordinate-contract` 在“有界需求坐标合同”范围内从
`partial` 升为 `verified`。后续遇到新的来源支持需求、来源或坐标漂移、反证、重复失败、
宿主或运行时变化，或有人提出残余缺口、Skill、Hook、标准主张时，按证据触发相关线复查。
